Web Scraping with Python

About the project

Summary:

Data scraping from the website: "www.boardgamegeek.com" and creation of four csv files about board games which create a small database. These four csv files contain information about the boardgame ratings, types, minimum and maximum number of players, price, their designers, publishers, artists etc. Total number of pages scraped: 25200.

Tools used:

Python with the following main libraries: Pandas, Beautiful Soup, Selenium

Project details:

In this project I used Python to scrape data from the most known and complete web site about board games. In this web site there is a ranking (list) of ALL board games ever created (including chess) spanning through almost 1490 pages of 100 board games each. I scraped the data up to page 252 (meaning a total number of 25200 pages) because from then on the games were not well known, did not have much info and thus were of no interest.

The data scraped have also been cleaned (in a degree) during the scraping procedure. Many difficulties across the creation of the appropriate code needed to be handled. The first of them was that the web site displays only the first 20 pages if you are not logged in. So I needed to automate the "sign in" procedure using "selenium" to automatically open the browser in the login page, insert my username and password and press the sign-in button. The second difficulty was that upon automatically opening the browser in the login page, an "accept cookies" pop-up appears in the bottom of the page interrupting the automatic completion of the username and password. Again using the "selenium" library an automatic "press ok to the cookie button" was made. Another difficulty was the missing values or missing elements in many categories such as some games have a standard number of players and not a minimum and a maximum number, which changes the structure of the page and a modification had to be made in the code.

After all issues were handled, the four parts of the code created four csv files. The first file (bg_database.csv), which is the main and biggest one, contains the table with the board game titles, Geek Rating(a rating computed by the websites algorithm), Average user rating, Number of voters, Price (amazon), Year, Complexity, Minimum number of players, Maximum number of players, Minimum time, Maximum time, Age, Type of the board game and Second type (if it the game has). The other three csv files (bg_designers, bg_publishers, bg_artists) contain information about the designers, the publishers and the artists of each board game. Keep in mind that in the case of designers, publishers and artists a maximum number of five was obtained as some games had many more, but obtaining everything from the web-site was not the goal of this project. All these four files create a relational database. The database creation and manipulation using SQL is the topic of the second Project.

Procedures - Tools

The main python libraries used are: "Pandas", "BeautifulSoup" and "Selenium". Other modules also used were the "time", "re" (regular expressions) and "requests".

  • The "Pandas" library is used to obtain the scraped information in the form of tables, manipulate them and finally produce the results in csv files.

  • The "requests" module is used to send HTTP requests using Python and helps us get the content of the web page as text.

  • The "BeatifulSoup" package is used for parsing and extracting data out of the HTML documents of the webpage, in which all the information is stored, by providing ways to navigate and search the parse tree.

  • The "Selenium" package helps us automate procedures of web pages such as logging in or pressing buttons.

  • The "re" module is used to help us obtain the desired information through text patterns.

  • The "time" module is just used to record the duration of the procedure.
